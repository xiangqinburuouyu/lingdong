## 步骤 1：上传工作流文件（2 分钟）

### 方式 A：使用 curl 命令（最快）

```bash
# 1. 设置你的 GitHub token（需要 workflow 权限）
export GITHUB_TOKEN=ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

# 2. 上传基础工作流
curl -X PUT \
  -H "Authorization: token $GITHUB_TOKEN" \
  -H "Accept: application/vnd.github.v3+json" \
  -d @- \
  "https://api.github.com/repos/xiangqinburuouyu/lingdong/contents/.github/workflows/build.yml" <<EOF
{
  "message": "Add GitHub Actions workflow",
  "content": "$(base64 -w 0 /home/admin/openclaw/workspace/temp/自媒体 APP/mobile-app/.github/workflows/build.yml)",
  "branch": "main"
}
EOF

# 3. 上传增强工作流
curl -X PUT \
  -H "Authorization: token $GITHUB_TOKEN" \
  -H "Accept: application/vnd.github.v3+json" \
  -d @- \
  "https://api.github.com/repos/xiangqinburuouyu/lingdong/contents/.github/workflows/build-advanced.yml" <<EOF
{
  "message": "Add advanced GitHub Actions workflow",
  "content": "$(base64 -w 0 /home/admin/openclaw/workspace/temp/自媒体 APP/mobile-app/.github/workflows/build-advanced.yml)",
  "branch": "main"
}
EOF
```

### 方式 B：GitHub 网页上传（无需 token）

1. 访问 https://github.com/xiangqinburuouyu/lingdong
2. 点击 **Add file** → **Create new file**
3. 文件名：`.github/workflows/build.yml`
4. 复制下方内容粘贴
5. 点击 **Commit changes**

**build.yml 内容**：
```yaml
name: Build Android APK

on:
  push:
    branches: [main]
  workflow_dispatch:

jobs:
  build:
    runs-on: ubuntu-latest
    timeout-minutes: 30

    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '18'
          cache: 'npm'
          cache-dependency-path: mobile-app/package-lock.json

      - name: Setup Expo
        uses: expo/expo-github-action@v8
        with:
          expo-version: latest
          eas-version: latest
          token: ${{ secrets.EXPO_TOKEN }}

      - name: Install dependencies
        working-directory: ./mobile-app
        run: npm ci

      - name: Build APK
        working-directory: ./mobile-app
        run: eas build --platform android --profile preview --non-interactive
        env:
          EXPO_TOKEN: ${{ secrets.EXPO_TOKEN }}

      - name: Notify build status
        run: |
          echo "✅ Build submitted!"
          echo "📱 Check: https://expo.dev/accounts/lindons/projects/selfmedia-app/builds"
```

## 步骤 2：配置 GitHub Secrets（2 分钟）

### 2.1 获取 Expo Token

1. 访问 https://expo.dev/settings/access-tokens
2. 点击 **Create Access Token**
3. 填写：
   - **Name**: `github-actions`
   - **Expiration**: 90 天
4. 点击 **Create**
5. **复制 token**（`exu_xxxxxxxxxx`）

### 2.2 添加到 GitHub

1. 访问 https://github.com/xiangqinburuouyu/lingdong/settings/secrets/actions
2. 点击 **New repository secret**
3. 填写：
   - **Name**: `EXPO_TOKEN`
   - **Value**: 粘贴 Expo token
4. 点击 **Add secret**

## 🆘 遇到问题？

### 问题 1：上传文件失败

**错误**: "403 Forbidden"

**解决**: 
```bash
# 确保 token 有 workflow 权限
# 访问 https://github.com/settings/tokens
# 创建新 token，勾选 repo + workflow
```

### 问题 2：Workflow 不显示

**解决**:
1. 检查文件路径：`.github/workflows/`
2. 检查 YAML 语法
3. 刷新 Actions 页面

### 问题 3：Build 失败

**错误**: "EXPO_TOKEN not found"

**解决**:
1. 确认 Secret 名称是 `EXPO_TOKEN`（大写）
2. 重新添加 Secret
3. 重新触发构建
