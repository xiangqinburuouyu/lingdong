## 步骤 1：上传工作流文件（2 分钟）

### 1.1 访问 GitHub 仓库

```
https://github.com/xiangqinburuouyu/lingdong
```

### 1.2 创建工作流文件

1. 点击 **Add file** → **Create new file**
2. 在文件名输入框中输入：`.github/workflows/build.yml`
   - 这会自动创建 `.github/workflows/` 目录
3. 复制下方**完整内容**粘贴到内容区域

### 1.3 工作流内容（复制这个）

```yaml
name: Build Android APK

on:
  push:
    branches: [main]
  workflow_dispatch:

jobs:
  build:
    runs-on: ubuntu-latest
    timeout-minutes: 30

    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '18'
          cache: 'npm'
          cache-dependency-path: mobile-app/package-lock.json

      - name: Setup Expo
        uses: expo/expo-github-action@v8
        with:
          expo-version: latest
          eas-version: latest
          token: ${{ secrets.EXPO_TOKEN }}

      - name: Install dependencies
        working-directory: ./mobile-app
        run: npm ci

      - name: Build APK
        working-directory: ./mobile-app
        run: eas build --platform android --profile preview --non-interactive
        env:
          EXPO_TOKEN: ${{ secrets.EXPO_TOKEN }}

      - name: Notify build status
        run: |
          echo "✅ Build submitted!"
          echo "📱 Check: https://expo.dev/accounts/lindons/projects/selfmedia-app/builds"
```

4. 滚动到页面底部
5. 点击 **Commit changes**

## 🆘 常见问题

### Q1: 看不到 "Build Android APK" 工作流？

**解决**:
1. 检查工作流文件是否正确上传
2. 访问 https://github.com/xiangqinburuouyu/lingdong/tree/main/.github/workflows
3. 确认 `build.yml` 文件存在
4. 刷新 Actions 页面

### Q2: 构建失败？

**查看错误日志**:
1. 点击失败的构建
2. 查看具体错误信息
3. 常见错误：
   - `EXPO_TOKEN not found` → 检查 Secret 名称是否正确
   - `Gradle build failed` → Expo 云端问题，重试即可

### Q3: 构建太慢？

**正常现象**:
- 首次构建需要 20-25 分钟
- 后续构建会更快（15-20 分钟）
- 取决于 Expo 队列长度
