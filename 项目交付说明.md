## 🚀 启动指南

### 第一步：启动 MongoDB

```bash
# 确保 MongoDB 已安装并运行
mongod --dbpath /path/to/data
```

或使用 Docker:
```bash
docker run -d -p 27017:27017 --name mongodb mongo:latest
```

### 第二步：启动后端

```bash
cd backend

# 安装依赖
npm install

# 配置环境变量
cp .env.example .env
# 编辑 .env 文件，设置 MongoDB 连接地址

# 启动服务
npm run dev
```

后端将在 `http://localhost:3000` 启动

### 第三步：启动前端

```bash
cd mobile-app

# 安装依赖
npm install

# 修改 API 地址（重要！）
# 编辑 src/config/api.js
# 将 API_BASE_URL 改为你的后端地址

# 启动 Expo
npx expo start
```

使用 Expo Go APP 扫描二维码即可在手机上运行。

## 📱 真机调试注意事项

### iOS/Android 真机

1. 确保手机和电脑在同一 WiFi 网络
2. 将 API 地址从 `localhost` 改为电脑的局域网 IP
   ```javascript
   // src/config/api.js
   const API_BASE_URL = 'http://192.168.x.x:3000/api';
   ```
3. 确保防火墙允许 3000 端口访问

### 模拟器

- iOS 模拟器 (Mac): 使用 `localhost` 即可
- Android 模拟器: 使用 `10.0.2.2` 代替 `localhost`

## 🧪 API 测试

### 使用 Postman/Insomnia

导入 API 文档中的接口进行测试。

### 使用 curl

```bash

curl -X POST http://localhost:3000/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{

    "password": "123456"

  -d '{
```

## ❓ 常见问题

### Q: MongoDB 连接失败？
A: 确保 MongoDB 服务正在运行，检查 `.env` 文件中的连接地址。

### Q: 前端无法连接后端？
A: 真机调试时需要使用电脑 IP 地址，不是 localhost。

### Q: Expo 扫码失败？
A: 确保手机和电脑在同一网络，检查防火墙设置。

### Q: 如何添加新功能？
A: 参考现有代码结构，在对应目录添加文件，保持代码风格一致。
